# Архитектура системы

## Выбранный вариант

Используется гибрид `Google Sheets + Apps Script + Python в GitHub Actions`.

| Вариант | Ежедневная работа | Надёжность 24/7 | Масштабирование | Стоимость | Поддержка |
|---|---|---:|---:|---:|---:|
| Только Apps Script | очень просто | средняя, жёсткие квоты | низкое | бесплатно | просто |
| Локальный Python | неудобно, компьютер должен работать | низкая | среднее | бесплатно | средне |
| Платный сервер | удобно | высокая | высокая | платно | сложно |
| Telegram-бот без таблицы | удобно для чтения | средняя | среднее | бесплатно | теряется рабочая таблица |
| Выбранный гибрид | привычная таблица и каналы | высокая для бесплатной схемы | высокая | бесплатно | умеренно |

Apps Script отвечает за привычный интерфейс, основной обход, Google News, личное меню бота и окончательную доставку. GitHub Actions даёт больше времени и памяти для Python, локальной модели, OCR и Instagram. Токен Telegram остаётся только в защищённых свойствах Apps Script.

По актуальной документации GitHub стандартные hosted runners бесплатны для публичных репозиториев: <https://docs.github.com/en/actions/concepts/billing-and-usage>. Для обычного Google-аккаунта Apps Script ограничен 90 минутами триггеров и 20 000 URL Fetch в сутки: <https://developers.google.com/apps-script/guides/services/quotas>. Поэтому тяжёлая обработка вынесена из Apps Script.

GitHub автоматически отключает расписания публичного репозитория после 60 дней без активности: <https://docs.github.com/en/actions/how-tos/manage-workflow-runs/disable-and-enable-workflows>. Чтобы владельцу не приходилось следить за этим вручную, отдельное ежемесячное расписание создаёт крошечное служебное обновление репозитория.

## Контуры

### Республиканский

Проверяет сайты, Google News, 41 республиканский Telegram-канал и YouTube. Берёт любое содержательное упоминание СКО. Региональные СМИ жёстко исключены из основного канала.

### Ежедневный мониторинг

Проверяет 32 городских и общественных Instagram-профиля, а также доступные Threads, VK, Facebook и TikTok-ссылки. Категории: дороги, ЖКХ, благоустройство, животные, паводок, транспорт, экология и происшествия.

### Региональные СМИ

Остаются отдельным контуром для позитива и ручного поиска за период. Они не могут попасть в республиканский Telegram-канал.

## Семантический анализ

Первый слой — высокополнотные правила на русском и казахском с учётом корней и контекста. Второй слой — бесплатная локальная модель `paraphrase-multilingual-MiniLM-L12-v2`. При недоступности модели мониторинг не падает и продолжает работу на первом слое.

Для изображений применяется Tesseract OCR. Для видео предусмотрены кадры через FFmpeg и локальный faster-whisper. Медиа скачиваются во временный каталог и удаляются сразу после анализа.

## Доставка и отказоустойчивость

Python передаёт находки в Apps Script порциями по 50 записей и отправляет heartbeat после каждого запуска. Временные ошибки повторяются с увеличивающейся задержкой. Релевантная ссылка попадает в техническую память Python только после подтверждения от Apps Script или постановки в локальную очередь Telegram. Поэтому не принятый Google пакет будет найден и передан повторно в следующем запуске.

Apps Script пишет строки в нужный лист и ставит сообщения в скрытую очередь. Тексты новостей постоянно не хранятся. Для защиты от повторов остаются только короткие ключи: нерелевантные 14 дней, доставленные релевантные ссылки 365 дней.

Состояния очереди: `pending`, `sending`, `retry`, `sent`, `skipped`. После ответа Telegram `200 OK` записываются URL-ключ и ключ `источник + заголовок`. При `429` используется `retry_after`, при сетевых ошибках — увеличивающаяся задержка.

Отказ одного источника не останавливает остальные. После трёх часов без heartbeat Apps Script сообщает владельцу, но основной Google-мониторинг продолжает работать.

## Границы бесплатной схемы

Это максимальная практическая надёжность без платной инфраструктуры, но не коммерческий SLA. GitHub может задержать плановый запуск, Instagram может потребовать обновить сессию, а удалённую до очередной проверки публикацию восстановить невозможно. Перекрытие 72 часа и независимый Apps Script заметно уменьшают риск пропуска.
