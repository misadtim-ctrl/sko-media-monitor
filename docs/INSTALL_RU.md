# Установка

Большую часть шагов можно выполнить автоматически. Ручными остаются только действия, где сервисы требуют подтверждение владельца.

## 1. Google Таблица

В существующий проект Apps Script добавляются два файла:

- `apps-script/Code.gs` вместо старого `Code.gs`;
- новый файл `apps-script/SourceRegistry.gs`.

После сохранения нужно обновить развёртывание веб-приложения, открыть таблицу и выбрать:

1. `Мониторинг СКО -> 1. Настроить таблицу`.
2. `Мониторинг СКО -> ВКЛЮЧИТЬ автопроверку`.
3. `Мониторинг СКО -> Подключить Python-помощник`.

Повторная настройка больше не очищает пользовательские источники и словари.

## 2. Два Telegram-канала с одним ботом

Основной канал остаётся прежним. Канал `Ежедневный мониторинг` создаётся тем же Telegram-аккаунтом. Того же бота нужно добавить в него администратором с правом публикации, затем выбрать в таблице `Настроить канал «Ежедневный мониторинг»` и указать `@имя_канала`.

Новый номер, новый Telegram-аккаунт и второй бот не нужны.

## 3. GitHub Actions

Репозиторий должен быть публичным, чтобы частые запуски стандартных GitHub-hosted runners не расходовали платные минуты. Секреты при этом остаются закрытыми.

Служебное расписание `Keep scheduled monitoring active` раз в месяц обновляет короткую строку в репозитории. Это предотвращает автоматическое отключение остальных расписаний после 60 дней без активности и не создаёт архив публикаций.

В `Settings -> Secrets and variables -> Actions` добавляются:

| Secret | Значение |
|---|---|
| `APPS_SCRIPT_WEBHOOK_URL` | адрес из пункта `Подключить Python-помощник` |
| `MONITOR_WEBHOOK_SECRET` | секрет из того же окна |
| `INSTAGRAM_USERNAME` | имя существующего Instagram-аккаунта |
| `INSTAGRAM_SESSION_B64` | содержимое `data/instagram-session.b64` |

`META_ACCESS_TOKEN` и `META_IG_USER_ID` необязательны. Они нужны только для официального Meta API.

## 4. Одноразовый вход в Instagram

На Mac дважды открыть:

1. `setup.command`.
2. `instagram-login.command`.

Имя, пароль и двухфакторный код вводятся во всплывающих окнах macOS без ограничения времени. Неверный пароль можно ввести повторно, а окно Terminal при ошибке остаётся открытым и показывает причину. Данные передаются непосредственно Instagram и не записываются в репозиторий. Готовая сессия хранится локально с ограниченными правами. Если GitHub уже авторизован на этом компьютере, тот же файл сам загрузит `INSTAGRAM_USERNAME` и зашифрованную сессию в GitHub Secret: ничего копировать вручную не нужно.

## 5. Проверка

В GitHub открыть вкладку `Actions` и вручную запустить оба процесса:

- `Monitor republican media`;
- `Monitor city complaints`.

После успешного запуска в таблице появится лист `ЕЖЕДНЕВНЫЙ МОНИТОРИНГ`, а новые релевантные ссылки начнут поступать в соответствующие каналы.

## Поиск старых материалов

В `Actions` выбрать `Search historical materials`, нажать `Run workflow`, ввести тему обычным языком, даты и область поиска. Python пройдёт доступные sitemap и WordPress-архивы, сравнит материалы с запросом локальной смысловой моделью и заменит содержимое листа `ПОИСК ЗА ПЕРИОД` новым результатом. Старые поиски в базе не накапливаются.
